# 功能规格写作示例库

> 用途：本文件用于给阶段 PRD 提供可复用的功能需求写作颗粒度。正式 PRD 不需要逐字照搬，但 P0 功能至少应达到接近以下示例的完整度。

## 1. 数据列表示例：对象列表

### FR-LIST-001 对象列表展示与选择

优先级：P0

用户问题：

- 用户导入多个模型后，需要知道当前项目里有哪些对象，并能快速选中目标对象进入编辑。

用户故事：

- 作为分镜创作者，我希望在左侧对象列表看到当前项目对象，以便快速定位并编辑模型。

入口：

- 工作台左侧资产面板的“对象”分组。

前置条件：

- 项目已初始化。
- 对象数据来自项目状态中的 `objects` 集合。

主流程：

1. 用户打开工作台。
2. 系统读取当前项目对象集合。
3. 系统按默认排序渲染对象列表。
4. 用户点击某一对象行。
5. 系统更新选中对象，并在右侧展示对象属性面板。

规格明细：

| 维度 | 说明 |
| --- | --- |
| 展示内容 | 对象名称、对象类型、可见状态、锁定状态、选中状态 |
| 数据来源 | 项目状态 `objects`；对象名称来自导入文件名或用户重命名值 |
| 数据规则 | 默认按导入时间升序；缺少名称时展示“未命名对象”；被删除对象不得继续出现在列表 |
| 交互规则 | 单击选中对象；点击可见开关切换显示；点击锁定开关切换编辑权限 |
| 状态规则 | 无对象展示空状态；加载失败展示错误状态；锁定对象行仍可选中但不可拖拽编辑 |
| 联动规则 | 选中对象后右侧面板切到对象属性；视口中对应模型高亮；时间线插帧目标切到该对象 |
| 持久化规则 | 列表本身不单独保存；对象状态由项目状态持久化或导出 |
| 性能约束 | 100 个对象内无需分页；超过 100 个对象需评估虚拟滚动或搜索 |

边界与异常：

- 无对象：展示空状态和导入入口，不展示空白列表。
- 无搜索结果：保留搜索词，展示“未找到匹配对象”，提供清空筛选入口。
- 对象被删除：如果删除的是当前选中对象，右侧面板回到世界属性或空选中状态。
- 对象锁定：行内显示锁定状态，禁止视口变换，但允许查看属性。

验收标准：

- AC-LIST-001：给定项目无对象，当用户打开对象列表，则展示空状态和导入入口。
- AC-LIST-002：给定项目存在 3 个对象，当用户打开对象列表，则按导入时间升序展示 3 条对象记录。
- AC-LIST-003：给定用户点击对象行，当点击完成，则该对象在列表、视口和右侧属性面板中保持同一选中状态。
- AC-LIST-004：给定当前选中对象被删除，当删除完成，则列表移除该对象，右侧面板不再展示已删除对象属性。

## 2. 表单/属性编辑器示例：对象参数调节

### FR-FORM-001 对象位置、旋转、缩放编辑

优先级：P0

用户问题：

- 用户需要精确调整模型在 3D 场景中的位置、朝向和大小，不能只依赖视口拖拽。

入口：

- 选中对象后的右侧对象属性面板“参数调节”分组。

前置条件：

- 当前选中对象存在。
- 对象未被锁定。

规格明细：

| 维度 | 说明 |
| --- | --- |
| 展示内容 | `position.x/y/z`、`rotation.x/y/z`、`scale.x/y/z` 数值输入；单位分别为米、度、倍数 |
| 数据来源 | 项目状态中的对象 transform；视口拖拽也会写入同一数据 |
| 数据规则 | 位置保留 2 位小数；旋转以度展示、内部可按弧度或度存储但需统一转换；缩放不得小于 0.01 |
| 交互规则 | 输入合法数值后实时更新状态和视口；非法输入回退到上一次合法值；对象锁定时控件禁用 |
| 状态规则 | 默认可编辑；锁定时只读；无选中对象时不展示对象参数；修改中不改变布局宽度 |
| 联动规则 | 数值输入和 TransformControls 拖拽双向同步；自动关键帧开启时按当前时间写入对应关键帧 |
| 持久化规则 | 写入项目状态；导出项目 JSON 时包含最新 transform |
| 性能约束 | 单次数值编辑应在一帧内更新视口，不触发模型重新加载 |

边界与异常：

- 输入空值：失焦时恢复上一次合法值。
- 输入非数字：不写入状态，并展示字段级错误。
- 对象锁定：控件禁用，提示“对象已锁定”。
- 自动关键帧开启：只有当前选中对象和当前时间满足插帧条件时才写入关键帧。

验收标准：

- AC-FORM-001：给定对象未锁定，当用户修改 `position.x`，则视口模型位置实时变化，项目状态同步更新。
- AC-FORM-002：给定对象已锁定，当用户查看参数调节，则位置、旋转、缩放输入控件不可编辑。
- AC-FORM-003：给定用户输入非法数值，当字段失焦，则字段恢复上一次合法值，不写入错误状态。

## 3. 上传/导入示例：GLB 文件导入

### FR-UPLOAD-001 导入 `.glb` 模型

优先级：P0

用户问题：

- 用户需要把已有 3D 模型放入分镜工作台，作为场景、角色或道具进行构图。

入口：

- 底部对象菜单“本地上传”。
- 空对象列表中的导入入口。

规格明细：

| 维度 | 说明 |
| --- | --- |
| 支持格式 | 仅支持 `.glb` |
| 文件限制 | 单文件导入；建议记录推荐大小和当前性能边界 |
| 数据来源 | 用户本地文件；运行时解析为 Three.js 场景对象和项目对象状态 |
| 解析流程 | 选择文件 -> 校验格式 -> 创建 object URL -> GLTFLoader 解析 -> 提取对象、材质、骨骼信息 -> 写入项目状态 |
| 成功反馈 | 模型出现在 3D 视口和对象列表，默认选中新导入对象 |
| 失败处理 | 格式不支持、解析失败、资源加载失败分别展示明确错误 |
| 清理策略 | 导入失败、对象删除、页面卸载时释放 object URL 和 Three.js 资源 |
| 联动规则 | 导入成功后对象列表新增记录，右侧面板切到对象属性，视口聚焦或高亮新对象 |

验收标准：

- AC-UPLOAD-001：给定用户选择合法 `.glb` 文件，当解析成功，则模型出现在视口和对象列表。
- AC-UPLOAD-002：给定用户选择非 `.glb` 文件，当系统校验格式，则阻止导入并提示仅支持 `.glb`。
- AC-UPLOAD-003：给定 `.glb` 解析失败，当失败发生，则不写入半成品对象，并展示失败提示。
- AC-UPLOAD-004：给定用户删除导入对象，当删除完成，则释放该对象关联的运行时资源。

## 4. 3D 视口示例：对象选择与拖拽

### FR-VIEWPORT-001 视口对象点选与变换

优先级：P0

用户问题：

- 用户在空间编辑时需要直接在 3D 视口选择对象并拖拽调整，而不是频繁回到列表。

规格明细：

| 维度 | 说明 |
| --- | --- |
| 可交互对象 | 项目对象、机位模型、骨骼节点、IK 目标点，具体优先级由当前编辑模式决定 |
| 选择规则 | 点击模型表面选中对象；点击空白区域清空对象选中或回到世界属性；播放预览中禁用编辑点选 |
| 拖拽规则 | 移动、旋转、缩放由当前工具模式决定；锁定对象不可拖拽 |
| 坐标与单位 | 位置使用世界坐标米制；旋转展示为度；缩放为倍率 |
| 状态规则 | 选中对象高亮；锁定对象可查看不可编辑；隐藏对象不可被点选 |
| 联动规则 | 视口选中同步列表和右侧面板；拖拽同步对象 transform；自动关键帧开启时写入时间线 |
| 性能约束 | 拖拽过程中不触发模型重新解析；辅助器刷新应与渲染帧同步 |

验收标准：

- AC-VIEWPORT-001：给定视口存在对象，当用户点击模型表面，则对象被选中并同步到列表和右侧面板。
- AC-VIEWPORT-002：给定对象未锁定且当前工具为移动，当用户拖拽移动控制器，则对象位置和右侧数值实时更新。
- AC-VIEWPORT-003：给定当前处于播放预览中，当用户点击视口对象，则不改变编辑选中状态。

## 5. 时间线示例：机位片段与关键帧

### FR-TIMELINE-001 机位片段轨与动画播放

优先级：P0

用户问题：

- 用户需要在一个动画预演中按时间切换不同机位，并检查对象、机位和骨骼关键帧的播放效果。

规格明细：

| 维度 | 说明 |
| --- | --- |
| 时间单位 | 支持帧和秒切换展示；内部以秒或帧统一换算 |
| 默认时长 | 由动画状态提供；若 UI 未开放编辑，PRD 必须明确写为未开放 |
| FPS | 默认 24，可编辑范围需写清；非法值不写入 |
| 轨道层级 | 顶部机位片段轨；下方汇总轨；对象、机位、骨骼详细通道按展开状态展示 |
| 片段规则 | 添加机位片段时使用当前播放头和当前机位；片段不得重叠；最短长度为 1 帧 |
| 关键帧规则 | 支持新增、选择、删除、拖动；聚合点编辑只影响其引用的关键帧 |
| 播放规则 | 播放时主视口进入纯净预览，按机位片段解析当前播放机位；结束后恢复编辑视角 |
| 联动规则 | 播放头移动同步视口采样；属性面板变更在自动关键帧开启时写入当前时间 |

验收标准：

- AC-TIMELINE-001：给定当前存在活跃机位，当用户点击“标记”，则在顶部机位轨生成从当前时间开始的机位片段。
- AC-TIMELINE-002：给定时间线上存在关键帧点，当用户选中并点击删除，则该关键帧从对应轨道中移除。
- AC-TIMELINE-003：给定机位片段覆盖当前播放时间，当用户播放动画，则主视口使用该片段对应机位进行预览。
- AC-TIMELINE-004：给定动画播放结束，当系统停止播放，则主视口恢复播放前的编辑视角。

## 6. 导出示例：快照导出

### FR-EXPORT-001 当前机位快照导出

优先级：P0

用户问题：

- 用户需要把当前机位画面导出为参考帧，用于分镜评审或后续 AI 视频生成参考。

规格明细：

| 维度 | 说明 |
| --- | --- |
| 导出入口 | 底部工具栏截图按钮；仅在存在当前机位时可用 |
| 导出对象 | 当前激活机位或选中机位的纯净渲染画面 |
| 文件格式 | PNG |
| 文件命名 | 使用项目名、机位名、时间戳组合；缺失时使用默认名称 |
| 数据来源 | Three.js 渲染结果、当前画幅比例、机位参数、世界背景 |
| 导出前校验 | 无机位时禁用；渲染未准备完成时展示生成中状态 |
| 成功反馈 | 生成快照记录并进入快照工作区；必要时触发下载 |
| 失败处理 | 渲染失败或浏览器限制时展示错误提示，不生成空记录 |
| 联动规则 | 快照卡片记录来源机位；“当前机位”筛选可筛出该快照 |

验收标准：

- AC-EXPORT-001：给定已选中机位，当用户点击截图，则生成 PNG 快照并进入快照工作区。
- AC-EXPORT-002：给定当前无可用机位，当用户查看截图按钮，则按钮禁用并说明原因。
- AC-EXPORT-003：给定选择了非默认画幅，当用户导出快照，则输出内容按画幅参考框裁切。
