# P0 基础编辑闭环开发方案

> 本文档用于确认 P0 的产品范围、技术方案、数据结构、验收标准和回滚策略。正式编码前，应先基于本文档完成确认。

## 1. 文档控制

- 产品名称：3D 导演台 / 轻量 3D 预演工作台
- 文档名称：P0 基础编辑闭环开发方案
- 文档版本：v0.1
- 文档状态：待确认
- 创建日期：2026-07-09
- 当前开发分支：`codex-p0-director-console-plan`
- 回滚基线：创建分支前的 `main` 当前提交

## 2. 一页摘要

P0 目标是补齐 3D 场景搭建最基础、最高频、最容易影响用户信任的编辑闭环，包括复制、多选、分组、搜索、撤销重做、右键菜单、吸附对齐和项目保存打开。

本阶段完成后，用户应能像使用一个轻量导演台一样，反复搭建、试错、整理、保存和恢复场景，而不是每次操作都担心误删、丢失或无法批量调整。

## 3. 背景与问题

当前产品已支持对象导入、插入、变换、材质、相机、世界、骨骼、时间线、快照和导出。但对于小白搭建镜头画面来说，基础编辑能力仍有明显断点：

- 只能单个对象编辑，无法多选和批量调整。
- 模型不能复制，重复布置道具、角色、群众和场景元素成本高。
- 不能打组，复杂场景难以整理。
- 缺少撤销重做，误操作成本高。
- 左侧搜索框已有视觉入口，但未形成完整过滤体验。
- 缺少项目保存打开，刷新或关闭页面后无法恢复工作。
- 缺少右键菜单和常见对象操作入口，很多操作隐藏在面板或列表按钮中。
- 吸附、落地和对齐能力不够直接，小白难以快速摆齐画面。

## 4. 产品目标

### 4.1 用户目标

- 快速复制已有模型、道具、相机或占位体。
- 一次选中多个对象做移动、隐藏、锁定、删除或成组。
- 把一组对象作为一个整体管理和移动。
- 操作错了可以撤销，撤销后也可以重做。
- 能搜索、整理、保存并重新打开项目。
- 能用右键完成高频对象操作。
- 能快速把对象放到地面、按网格或对象边界对齐。

### 4.2 产品目标

- 建立后续模板、构图、镜头板和 AI 交付的编辑底座。
- 降低小白用户搭场景时的挫败感。
- 保持现有单选、相机、骨骼、时间线和导出功能不回退。

## 5. 本次范围

### 5.1 对象复制与粘贴

功能要求：

- 支持复制当前选中的对象。
- 支持粘贴为新对象，新对象拥有独立 `id` 和名称。
- 粘贴对象默认有轻微位置偏移，避免完全重叠后用户以为没成功。
- 支持复制对象的基础属性：名称、类型、可见性、锁定状态、包围盒状态、变换、材质覆盖、骨骼 Pose、IK 链目标。
- 对导入 GLB 对象，复制对象复用同一个 `assetId`，不重复创建资产记录。
- 对内置对象和几何体，复制结果应保持原外观。

第一版边界：

- 不复制时间线关键帧。
- 不复制快照记录。
- 不复制运行时 Three.js 对象实例，只复制项目状态，再由视口同步生成运行时对象。

### 5.2 多选与批量操作

功能要求：

- 支持在左侧列表和视口中多选对象。
- 支持 `Shift` 追加选择、`Meta/Ctrl` 切换选择。
- 多选时右侧显示“多选属性”面板。
- 多选对象可批量隐藏、显示、锁定、解锁、删除、成组。
- 多选对象可通过 TransformControls 作为一个临时整体移动、旋转和缩放。

第一版边界：

- 多选只覆盖对象，不覆盖相机和骨骼控制点。
- 多选状态下不编辑材质、骨骼和单对象尺寸。

### 5.3 分组与解组

功能要求：

- 支持把多个对象打成组。
- 支持组整体选中、移动、旋转、缩放、隐藏、锁定和删除。
- 支持解组，解组后对象保持当前世界位置不跳变。
- 左侧资产列表展示组层级。
- 组可重命名。

数据原则：

- 分组应进入项目数据，而不是只在 Three.js 场景里临时创建。
- 对象保留自身变换，组拥有独立变换。
- 渲染时需要正确计算组变换和子对象变换。

第一版边界：

- 暂不支持组内组的无限复杂编辑，可预留数据结构，但 UI 第一版只保证一层组稳定可用。
- 暂不支持相机入组。

### 5.4 左侧搜索

功能要求：

- 左侧搜索框可过滤对象和机位。
- 搜索对象名称、机位名称和类型标签。
- 搜索状态下保留当前选中项可见。
- 无结果时展示空状态。

第一版边界：

- 不做高级筛选语法。
- 不做跨快照、时间线、材质名搜索。

### 5.5 撤销与重做

功能要求：

- 支持撤销和重做项目状态变化。
- 覆盖对象新增、复制、粘贴、删除、变换、显隐、锁定、重命名、分组、解组、世界属性、相机属性等 P0 相关操作。
- 支持快捷键：`Meta/Ctrl+Z` 撤销，`Meta/Ctrl+Shift+Z` 或 `Meta/Ctrl+Y` 重做。
- 顶部或工具区提供撤销/重做按钮，并在不可用时禁用。

技术原则：

- 使用状态快照或 action patch 记录历史。
- 需要限制历史栈长度，避免大项目内存无限增长。
- 运行时资源如 object URL、Three.js 对象不应直接进入历史栈。

第一版边界：

- 不要求撤销视频导出或文件下载动作。
- 不要求恢复已释放的外部 object URL；涉及项目保存时另行处理资产引用。

### 5.6 右键菜单

功能要求：

- 在视口对象和左侧对象列表上支持右键菜单。
- 菜单项包括：复制、粘贴、重命名、隐藏/显示、锁定/解锁、删除、成组、解组、吸附到地面。
- 菜单根据当前选择状态启用或禁用。
- 右键空白区域可提供粘贴、清除选择、视角重置等入口。

第一版边界：

- 不做复杂嵌套菜单。
- 不做自定义快捷键配置。

### 5.7 吸附到地面与对象对齐

功能要求：

- 支持将选中对象或多选对象吸附到地面。
- 支持按 X / Y / Z 轴对齐对象中心。
- 支持按底部、顶部、左侧、右侧、前后边界对齐。
- 支持批量分布：横向等距、纵向等距。
- 与现有网格吸附设置保持一致，不破坏世界属性中的 snap 配置。

第一版边界：

- 不做物理碰撞。
- 不做复杂表面吸附，只做地面和包围盒对齐。

### 5.8 项目保存、打开与自动保存

功能要求：

- 支持导出项目文件，建议扩展名为 `.3dwb.json`。
- 支持打开项目文件并恢复项目状态。
- 支持本地自动保存到 `localStorage` 或 `IndexedDB`。
- 页面加载时如存在自动保存内容，应提示恢复。
- 项目文件包含场景对象、相机、世界设置、材质覆盖、时间线、机位片段、快照元数据。

资产处理原则：

- 对已导入的 GLB、贴图和全景图，第一版项目文件可记录资产元数据和 object URL 不可恢复提示。
- 若要完整恢复导入资产，需要后续引入文件句柄、IndexedDB Blob 存储或资源打包方案。

第一版建议：

- 本轮先实现“项目 JSON 保存 / 打开基础状态”。
- 对本地导入资产，在打开项目时给出缺失资产提示，并保留对象占位或要求重新绑定。
- 自动保存优先保存可序列化项目状态，不保存大文件 Blob。

## 6. 非范围

P0 不做以下能力：

- 场景模板、镜头模板和姿势库。
- Shot Board。
- AI 视频提示词生成。
- 云端项目同步。
- 多人协作。
- 完整资产打包格式。
- 多格式模型导入。
- 高级灯光、物理、碰撞和材质节点系统。

## 7. 建议技术方案

### 7.1 状态结构调整

建议在 `ProjectState` 中新增：

- `selectedObjectIds: string[]`
- `groups: SceneGroup[]`
- `clipboard?: ProjectClipboard`
- `history: ProjectHistoryState`

建议新增类型：

```ts
export type SceneGroup = {
  id: string;
  name: string;
  visible: boolean;
  locked: boolean;
  objectIds: string[];
  position: Vec3;
  rotation: Vec3;
  scale: Vec3;
};

export type ProjectClipboard = {
  kind: "objects";
  objectIds: string[];
  objects: SceneObject[];
  groups: SceneGroup[];
  copiedAt: string;
};

export type ProjectHistoryState = {
  past: ProjectStateSnapshot[];
  future: ProjectStateSnapshot[];
  limit: number;
};
```

注意：实际实现时不要把 `history` 自己嵌套进快照，避免无限递归和大对象膨胀。

### 7.2 Store action 拆分

建议新增或扩展以下 action：

- `setSelectedObjects`
- `toggleSelectedObject`
- `selectObjectRange`
- `duplicateSelection`
- `copySelection`
- `pasteClipboard`
- `groupSelection`
- `ungroupSelection`
- `renameGroup`
- `removeSelection`
- `setSelectionVisible`
- `setSelectionLocked`
- `alignSelection`
- `snapSelectionToGround`
- `undo`
- `redo`
- `saveProjectSnapshot`
- `loadProjectSnapshot`

### 7.3 渲染层方案

- 视口层根据 `selectedObjectIds` 渲染多选高亮。
- 多选 TransformControls 可绑定到临时 pivot group。
- 变换结束后，将临时 group 的矩阵增量分发给各对象。
- 组对象在 Three.js 中以 `Group` 表示，但项目状态仍是权威来源。
- 打组/解组时必须保持世界矩阵不跳变。

### 7.4 项目保存方案

- 新增 `src/export/projectExport.ts` 处理项目序列化、反序列化、文件下载。
- 新增 `src/domain/projectSerialization.ts` 处理 schema version、默认值补齐和兼容性迁移。
- 自动保存逻辑建议放在 `App` 或独立 hook 中，不放进 3D 渲染循环。

## 8. 开发拆分建议

虽然 P0 是一个完整阶段，但建议按以下顺序开发和验收：

1. 分支与文档确认。
2. 状态结构升级：多选、组、剪贴板、历史栈的数据类型。
3. 左侧搜索、单选兼容和多选选择逻辑。
4. 复制、粘贴、删除确认。
5. 撤销和重做。
6. 打组、解组和左侧层级展示。
7. 多选 TransformControls 和组变换。
8. 右键菜单。
9. 吸附到地面和对齐分布。
10. 项目保存、打开和自动保存。
11. 回归测试：现有导入、相机、骨骼、时间线、快照和导出。

## 9. 验收标准

### 9.1 基础编辑

- 选中一个对象后复制粘贴，新对象出现在场景中，名称和位置可区分。
- 复制导入 GLB 对象后，新对象正常渲染，材质覆盖保留。
- 删除对象前有确认或可通过撤销恢复。
- 左侧搜索输入对象名称后，只展示匹配对象。

### 9.2 多选与分组

- 用户可以通过列表和视口完成多选。
- 多选对象可以批量隐藏、锁定、删除。
- 多选对象可以成组，组在左侧层级中可见。
- 移动组后，子对象相对位置保持一致。
- 解组后，子对象世界位置不变化。

### 9.3 撤销重做

- 对象变换后可撤销，再重做。
- 复制、删除、成组、解组均可撤销。
- 连续多步操作的历史顺序正确。
- 撤销/重做不破坏相机、时间线和骨骼状态。

### 9.4 对齐吸附

- 选中对象可一键落到地面。
- 多选对象可按指定轴中心对齐。
- 多选对象可等距分布。

### 9.5 项目保存

- 用户可导出项目 JSON。
- 用户可打开项目 JSON 并恢复对象、相机、世界设置和时间线基础状态。
- 页面刷新后可提示恢复自动保存内容。
- 缺失本地导入资产时有明确提示，不导致页面崩溃。

### 9.6 回归

- GLB 导入仍可用。
- 材质编辑仍可用。
- 相机创建、预览、目标绑定和截图仍可用。
- 世界属性、全景球和网格吸附仍可用。
- FK / IK 基础骨骼编辑仍可用。
- 时间线关键帧、机位片段和视频导出入口仍可用。

## 10. 风险与处理

| 风险 | 影响 | 处理建议 |
| --- | --- | --- |
| 多选 TransformControls 与现有骨骼控制冲突 | 选中逻辑混乱 | 多选仅对象层级生效，骨骼控制点不进入多选 |
| 分组后世界矩阵跳变 | 场景位置错误 | 打组/解组必须用矩阵换算保持世界变换 |
| 历史栈记录过大 | 内存膨胀 | 限制历史长度，排除运行时对象和大资源 |
| 导入资产无法完整保存 | 打开项目后模型缺失 | 第一版明确提示缺失资产，后续做 IndexedDB Blob 或项目打包 |
| P0 范围较大 | 开发回归风险高 | 按原子功能逐步提交和验收 |

## 11. 回滚与分支策略

当前已从 `main` 创建独立分支：

- 分支名：`codex-p0-director-console-plan`
- 目的：保存 P0-P4 路线图、P0 方案和后续 P0 开发代码。

回滚策略：

1. 本次所有 P0 文档和代码只在 `codex-p0-director-console-plan` 分支推进。
2. 如果开发效果不好，可以直接切回 `main`，不合并该分支。
3. 如果只想回滚 P0 代码但保留方案文档，可在该分支上按提交粒度 revert。
4. 正式进入 P0 编码后，建议按功能提交：搜索、多选、复制、撤销、分组、右键、对齐、保存分别提交。
5. 每个原子功能完成后运行构建和基础回归，避免最后一次性排查。
