## 2. 一页摘要

### 一句话结论

当前版本已经形成一个可用的 3D 影视分镜预演工作台，支持 `.glb` 模型导入、对象与摄影机编辑、骨骼 `FK / IK` 静态 Pose 调整、世界环境设置、画幅切换和机位快照导出。

### 本次文档覆盖的产品范围

- 左侧资产列表：对象、机位、导入入口
- 中间 3D 视口：场景浏览、选中、拖拽编辑、摄影机预览
- 右侧属性区：世界 / 对象 / 机位属性与快照工作区
- 底部工具条：移动模式、对象插入、全景图、机位、画幅、截图

### 当前版本已具备的核心能力

- 仅支持 `.glb` 文件导入
- 对象移动、旋转、缩放、隐藏、锁定、删除
- 材质颜色、粗糙度、金属度、透明度、贴图替换
- 摄影机创建、视角切换、`FOV` 调整、自由朝向 / 注视目标
- 世界级场景参数、全景球、网格吸附、地面控制
- 带骨骼模型的 `FK / IK` 静态编辑
- 快照导出与快照列表管理
- 常用画幅比例切换与截图联动

### 当前版本不包含

- 关键帧时间线
- 本地项目保存 / 打开项目
- 多镜头时间轴与镜头编排
- 动画片段导入、播放、混合
- AI 视频平台接口直连
- 多格式导入（如 `fbx / obj / usd / gltf`）

### 当前版本最大限制

- 数据仅保存在内存
- 刷新页面后不保留项目状态
- 骨骼控制以静态 Pose 为主，不支持动画时间线
- `IK` 以当前模型可识别骨链和运行时控制点为基础，不是完整 DCC 级绑定系统

## 3. 背景、问题与依据

### 背景

产品面向影视分镜预演与 AI 视频前置构图阶段，目标不是替代 Blender、Maya、Unreal 等重型工具，而是提供一个更轻、更快、更贴近分镜协作心智的 3D 工作台。

### 用户问题

- 仅靠提示词难以稳定表达机位、对象空间关系和镜头连续性
- 重型 3D 软件学习成本高，不适合快速出镜头参考
- AI 视频创作前，用户需要快速验证构图、人物站位、镜头朝向和画幅
- 导入模型后，常见高频动作包括摆位、改材质、加机位、截图、调骨骼 Pose

### 产品判断

当前版本优先保证“轻量预演闭环”而不是“完整 3D 创作闭环”，即：

1. 先解决导入、摆位、机位、截图和 Pose 这些最常见动作。
2. 再逐步扩展到时间线、镜头编排、项目持久化等更重的能力。

## 4. 目标用户、场景与用户旅程

### 用户角色

| 用户类型 | 目标 | 痛点 | 使用频率 |
| --- | --- | --- | --- |
| 导演 / 分镜创作者 | 快速搭景、定机位、出参考帧 | 空间和镜头试错成本高 | 高频 |
| AI 视频创作者 | 在生成前稳定镜头和构图 | 纯文本很难精确控制空间 | 高频 |
| 预演执行 / 3D 协作人员 | 快速准备可沟通的站位和镜头参考 | 重型软件流程过长 | 中高频 |

### 使用场景

- 导入角色、道具或环境模型，快速摆放对象关系
- 创建多个机位并对比构图
- 为角色调整静态动作 Pose
- 切换不同画幅查看镜头输出边界
- 导出当前机位参考帧，用于沟通、评审或后续 AI 视频生成

### 用户旅程

| 步骤 | 用户行为 | 用户目标 | 系统响应 |
| --- | --- | --- | --- |
| 1 | 导入 `.glb` 或插入内置对象 | 搭建基础场景 | 资产进入对象列表并出现在视口 |
| 2 | 调整对象位置 / 旋转 / 缩放 / 材质 | 完成空间关系和视觉预演 | 右侧对象属性与视口实时同步 |
| 3 | 创建或选择摄影机 | 确认镜头构图 | 视口与机位预览联动 |
| 4 | 如有角色骨骼，进入 `FK / IK` | 调整人物静态姿态 | 视口显示骨骼辅助与控制点 |
| 5 | 调整世界、全景球、地面与网格 | 完成环境基准设置 | 世界属性即时生效 |
| 6 | 切换画幅并截图 | 产出参考帧 | 快照进入快照工作区 |

## 5. 产品目标、非目标与成功标准

### 产品目标

- 建立影视分镜预演所需的最小 3D 编辑闭环
- 降低镜头预演与构图沟通成本
- 为后续时间线和项目制协作打下稳定数据基础

### 体验目标

- 大多数高频动作都能在一个屏幕内完成
- 左侧管资产，中间做编辑，右侧看上下文属性，底部放快捷动作
- 视口操作和右侧参数编辑保持一致，不出现“双轨状态”

### 非目标

- 不做完整建模软件
- 不做完整动画软件
- 不做本地持久化项目系统
- 不做复杂渲染、灯光、物理和特效

### 当前版本成功标准

| 指标 | 类型 | 目标值或观察方式 | 是否验收项 |
| --- | --- | --- | --- |
| `.glb` 导入成功 | 定性 | 用户能导入合法 `.glb` 并在视口看到对象 | 是 |
| 对象编辑闭环 | 定性 | 位置、旋转、缩放、材质编辑都能立即反映到视口 | 是 |
| 摄影机编辑闭环 | 定性 | 能创建机位、调整 `FOV`、切换预览并截图 | 是 |
| 骨骼静态编辑 | 定性 | 带骨骼模型支持 `FK / IK` 基础操作 | 是 |
| 快照导出闭环 | 定性 | 已选中机位时可导出快照并写入快照列表 | 是 |

## 8. 功能需求与规则

### 8.1 场景与视口基础能力

用户故事：

- 作为创作者，我希望能在 3D 视口中浏览场景、选择对象并进行拖拽编辑，以便快速完成预演搭建。

规则：

- 视口支持环绕、缩放、平移
- 显示网格地面与方向辅助
- 支持视口点击选择对象或机位
- 选中目标后，右侧面板自动切换到对应属性
- 视口空白区域点击后，回到世界属性上下文

验收标准：

- 给定场景内存在对象和机位，当用户点击对应元素，则右侧展示对应上下文面板。

### 8.2 `.glb` 导入与资源生命周期

用户故事：

- 作为创作者，我希望导入本地 `.glb` 模型并直接开始编辑，以便快速把参考资产放入工作台。

规则：

- 仅支持 `.glb`
- 导入后自动进入对象列表并在视口可见
- 导入失败时展示错误信息
- 删除对象、替换资源或页面卸载时需要释放运行时资源与 `objectURL`
- 当前不做本地项目保存

验收标准：

- 给定合法 `.glb` 文件，当用户导入后，则对象可见、可选中、可编辑。

### 8.3 左侧资产列表

用户故事：

- 作为创作者，我希望在左侧集中管理对象和机位，以便快速切换编辑目标。

规则：

- 对象区显示所有对象
- 机位区显示所有机位
- 每一行支持隐藏 / 锁定 / 删除
- 当前选中项高亮
- 长名称需要截断，操作按钮区必须稳定可见

验收标准：

- 给定对象或机位存在，当用户点击行项，则切换当前上下文；点击行内图标，则执行对应操作且不触发行级选中冲突。

### 8.4 对象编辑

用户故事：

- 作为创作者，我希望既能在视口里拖动对象，也能在右侧输入精确参数，以便粗调和精调都顺手。

规则：

- 支持移动、旋转、缩放
- 视口 gizmo 与右侧参数联动
- 对象支持隐藏 / 锁定 / 删除
- 锁定后不可通过视口继续编辑
- 参数调节区以二级面板呈现
- 参数调节数值显示到两位小数

验收标准：

- 给定对象已选中，当用户拖动 gizmo 或输入数值，则对象变换与右侧参数保持同步。

### 8.5 对象实际尺寸与包围盒

用户故事：

- 作为创作者，我希望看到对象当前的实际尺寸结果，而不仅仅是缩放倍数，以便判断比例是否合理。

规则：

- 实际尺寸字段使用 `X / Y / Z`
- 单位固定为 `m`
- 精度到毫米级
- 按包围盒计算
- 结果为只读展示
- 支持显示 / 关闭包围盒

验收标准：

- 给定对象已选中，当对象缩放或姿态变化后，实际尺寸结果实时更新。

### 8.6 材质编辑

用户故事：

- 作为创作者，我希望快速调整材质表现，以便完成基础视觉探索。

规则：

- 读取对象材质槽
- 支持颜色、粗糙度、金属度、透明度
- 支持替换贴图
- 当前版本只做运行时覆盖，不做复杂材质节点系统

验收标准：

- 给定对象包含可编辑材质，当用户修改参数后，则视口材质效果即时变化。

### 8.7 摄影机管理与机位编辑

用户故事：

- 作为创作者，我希望管理多个机位并在它们之间切换，以便快速比较不同镜头方案。

规则：

- 支持新增机位
- 支持从当前视口创建机位
- 支持机位名称、位置、目标、`FOV`
- 支持显示 / 锁定 / 删除
- 支持进入摄影机预览
- 机位属性区包含实时预览窗口

验收标准：

- 给定至少一个机位存在，当用户调整机位参数后，则预览窗口和视口辅助状态同步更新。

### 8.8 镜头朝向与注视目标

用户故事：

- 作为创作者，我希望既能自由控制镜头，也能让镜头稳定注视某个对象或手动坐标，以便适配不同镜头需求。

规则：

- 镜头朝向支持 `自由朝向 / 注视目标`
- 注视目标支持两种来源：
  - 资产引用：对象或其他机位
  - 手动坐标：`X / Y / Z`
- 资产目标选择器使用 Suggest 一体化搜索下拉
- 占位文案统一为“搜索对象”
- `LookAt` 模式下，旋转不是主编辑入口，朝向由目标驱动

验收标准：

- 给定机位处于 `LookAt` 模式，当用户切换目标对象或修改手动坐标，则机位朝向即时更新。

### 8.9 世界属性与全景球

用户故事：

- 作为创作者，我希望在未选中具体元素时编辑整个 3D 世界，以便统一设定预演环境。

规则：

- 未选中对象 / 机位时展示世界属性面板
- 世界属性包括：
  - 场景整体位置 / 旋转 / 缩放
  - 全景球绑定与显示
  - 全景球半径
  - 水平旋转
  - 元素标签显示
  - 网格吸附开关与参数
  - 地面显示、高度、透明度
- 场景整体变换作用于世界根节点，不直接改变机位数据

验收标准：

- 给定无对象和机位选中，当用户修改世界属性，则主视口环境即时更新。

### 8.10 底部工具条

用户故事：

- 作为创作者，我希望通过底部栏快速切换模式和插入内容，以便减少面板跳转。

规则：

- `移动` 菜单支持：
  - 移动 `V`
  - 旋转 `R`
  - 缩放 `S`
- `对象` 菜单支持：
  - 本地上传
  - 男性素体
  - 女性素体
  - 群众（行数 x 列数 + 间距）
  - 几何模型：立方体、球体、圆柱体、环状体、圆锥、棱锥
- `全景图`：绑定全景图
- `添加机位`：快速新建机位
- `画幅比例`：
  - 默认
  - `21:9`
  - `16:9`
  - `4:3`
  - `1:1`
  - `3:4`
  - `9:16`
  - `2:3`
  - `3:2`
- `截图`：仅选中机位时可用

验收标准：

- 给定未选中机位，当用户尝试截图，则截图按钮应禁用。

### 8.11 快照工作区

用户故事：

- 作为创作者，我希望截图结果集中管理，并知道每一张来自哪个机位，以便做镜头对比和选片。

规则：

- 右侧提供 `属性 / 快照` tabs
- 快照卡片显示名称、机位来源、生成时间
- 支持 `全部 / 当前机位` 筛选
- 快照基于当前机位与当前画幅导出

验收标准：

- 给定机位已选中且完成截图，当用户切到快照 tab，则能看到新增快照记录。

### 8.12 骨骼控制

用户故事：

- 作为创作者，我希望对带骨骼模型做静态姿态编辑，以便完成站姿、动作和镜头关系调整。

规则：

- 自动识别骨架数据
- 支持骨架辅助显示开关
- 支持 `FK / IK` 两种编辑模式
- `FK`：
  - 默认选中根骨骼
  - 通过 Suggest 搜索框选择骨骼
  - 骨骼名称以中文友好名称展示，并保留原始名作为辅助识别
  - 当前版本主要支持骨骼旋转
- `IK`：
  - 以模型已有或自动生成的骨链控制点为主
  - 进入 `IK` 模式时显示控制点
  - 当前以点击视口中的 `IK` 节点后拖动调整为主
- 对象整体移动、旋转、缩放时，`IK` 控制点需保持与模型当前相对关系同步

验收标准：

- 给定带骨骼模型已导入，当用户在 `FK` 或 `IK` 模式下编辑后，则模型姿态立即变化且交互目标与当前模式一致。

## 9. 状态与交互规则总览

### 默认状态

- 默认显示世界属性
- 默认存在一个初始相机和一个占位角色
- 默认画幅为“默认”

### 选中状态

- 选中对象：进入对象上下文
- 选中机位：进入机位上下文
- 选中骨骼 / `IK` 点：对象上下文下进入骨骼编辑子状态

### 禁用状态

- 未选中机位时，截图按钮禁用
- 锁定对象 / 机位后，不允许在视口中直接编辑

### 空状态

- 无快照时，快照面板展示空状态
- 某些对象无骨架时，骨骼控制区展示说明文案
- 无匹配注视目标时，Suggest 展示空结果

### 异常状态

- 导入非 `.glb` 文件时，提示仅支持 `.glb`
- 资源释放或对象删除后，不保留悬挂引用

## 10. 数据结构与存储口径

### 存储策略

- 当前版本仅使用前端内存态
- 不做本地工程保存
- 刷新页面后不保留数据
- 必须在删除对象、替换贴图或页面关闭时释放运行时资源

### 核心数据对象

#### 资产 `AssetRecord`

- 类型：`glb / texture / panorama`
- 字段：`id / name / objectUrl / size / createdAt / mimeType`

#### 场景对象 `SceneObject`

- 基础字段：`id / name / type / visible / locked`
- 变换字段：`position / rotation / scale`
- 展示字段：`boundsVisible / actualDimensions`
- 材质字段：`materialOverrides`
- 骨骼字段：`rig`

#### 摄影机 `SceneCamera`

- 基础字段：`id / name / visible / locked`
- 变换字段：`position / rotation`
- 构图字段：`target / fov / mode`
- 目标引用：`targetMode / targetRefId / targetRefType`

#### 世界设置 `WorldSettings`

- 场景整体变换
- 标签显示
- 网格吸附
- 地面参数
- 全景球参数

#### 快照 `SnapshotRecord`

- `id / name / cameraId / createdAt / imageDataUrl`

## 11. 技术方案摘要

### 前端技术栈

- Vite
- React
- TypeScript
- Three.js
- Zustand

### 模块分层

- `src/components`：界面层
- `src/three`：Three.js 运行时与场景对象管理
- `src/store`：状态中心
- `src/domain`：产品数据结构与规则
- `src/export`：快照导出

### 设计原则

- UI、状态、Three.js 运行时解耦
- 高风险交互以运行时对象和 store 双向同步为核心
- 所有新增功能尽量拆成原子能力，便于后续扩展
- 项目规范中优先使用 `elementdesign` 组件库；当前未全面接入前，可采用轻量自定义实现

## 13. 已知限制与风险

- 当前没有项目级持久化，适合快速预演，不适合长周期工程管理
- 骨骼 `IK` 仍偏工作台级能力，不是完整角色动画系统
- 贴图与大模型导入对浏览器内存敏感
- 快照以图片数据保存在内存，长时间大量使用会增加内存占用
