# 3D 影视分镜工作台产品需求文档

## 1. 文档信息

- 产品名称：3D 影视分镜工作台
- 文档版本：v0.1
- 文档状态：讨论稿
- 创建日期：2026-06-22
- 负责人：待定
- 开发约定：本项目每次开发都必须先更新或引用本需求文档，再进行代码修改。

## 2. 产品定位

3D 影视分镜工作台是一个面向影视分镜预演和 AI 视频创作流程的轻量级 3D 虚拟摄影工具。

产品帮助创作者导入 `.glb` 三维模型，进行基础摆放、材质颜色调整、摄影机关键帧设计，并导出参考帧和结构化镜头参数，用于后续视觉沟通、分镜预演或 AI 视频生成。

本产品不是完整的 3D 建模软件，也不是 Blender 的替代品。第一版应更像一个“轻量虚拟摄影台”：让导演、分镜师和 AI 视频创作者快速确认空间关系、构图和镜头运动。

## 3. 目标用户

### 主要用户

- 需要快速做镜头预演的导演和分镜师。
- 需要稳定空间参考的 AI 视频创作者。
- 制作概念片、预告片、广告片或提案视频的小型创意团队。

### 次要用户

- 需要为非 3D 协作者准备机位参考的 3D 创作者。
- 需要可控构图和镜头参数的产品或场景可视化团队。

## 4. 用户问题

1. AI 视频工具擅长生成，但对空间关系、机位和对象位置的精确控制较弱。
2. 仅靠文字提示词难以稳定保持镜头位置、对象布局和多镜头连续性。
3. 传统 3D 软件功能过重，不适合快速分镜预演。
4. 创作者需要可复用的参考帧和镜头参数，而不是一次性的截图。
5. 材质颜色是视觉探索中的高频需求，但第一版不需要完整材质编辑器。

## 5. 产品目标

### MVP 目标

- 支持导入 `.glb` 模型到 3D 场景。
- 支持基础模型摆放：移动、旋转、缩放、隐藏、复制、删除。
- 支持静态材质基础颜色调整。
- 支持带骨骼模型的静态 Pose 编辑。
- 支持创建和管理影视摄影机。
- 支持摄影机和模型对象的关键帧时间线。
- 支持在视口中预览镜头运动。
- 支持导出参考帧和结构化镜头参数。

### MVP 非目标

- 不做网格、顶点、UV 编辑。
- 不做角色蒙皮权重、骨骼绑定关系重建或自动绑定。
- 不做完整角色动画时间线编辑。
- 不做程序化材质节点编辑器。
- 不做物理模拟。
- 不直接接入 AI 视频生成接口。
- 不做多人协作。
- 第一版不支持 `.obj`、`.fbx`、`.gltf`、`.usd` 等格式导入。

## 6. 已确认产品决策

- 使用场景：影视分镜预演。
- 导入格式：仅支持 `.glb`。
- 输出内容：参考帧和摄影机/镜头参数。
- 材质能力：需要支持静态材质颜色调整。
- 时间线能力：需要关键帧时间线。

## 7. MVP 功能需求

### 7.1 GLB 导入

用户可以导入一个或多个 `.glb` 文件。

功能要求：

- 支持通过本地文件选择器导入 `.glb` 文件。
- 导入后模型显示在 3D 视口中。
- 新导入模型自动居中。
- 尽可能自动缩放到适合编辑和观察的默认尺寸。
- 尽可能让模型底部贴近地面网格。
- 导入模型后自动加入对象/图层列表。
- 保留模型层级结构，至少要能遍历并读取材质。

验收标准：

- 合法 `.glb` 文件导入后可以在场景中看到。
- 用户可以选中导入后的模型。
- 模型出现在对象/图层列表中。
- 过大或过小的模型会被归一化到可观察、可编辑的范围内。

### 7.2 3D 视口导航

用户可以在 3D 视口中检查场景。

功能要求：

- 支持环绕、平移、缩放视图。
- 显示地面网格。
- 显示世界坐标轴或方向辅助器。
- 提供基础场景灯光。
- 在技术可控的前提下支持阴影开关。

验收标准：

- 用户可以围绕导入模型自由观察。
- 用户可以缩放视图，且不容易丢失场景。
- 地面网格能提供清晰的空间参考。

### 7.3 模型变换编辑

用户可以调整模型位置、角度和尺寸。

功能要求：

- 支持在视口或对象列表中选中模型。
- 支持移动、旋转、缩放选中模型。
- 提供位置、旋转、缩放的数值输入。
- 支持重置变换。
- 支持复制模型。
- 支持删除模型。
- 支持隐藏/显示模型。
- 支持锁定/解锁模型，避免误操作。

验收标准：

- 变换修改能立刻反映在 3D 视口中。
- 数值输入和视口操控状态保持同步。
- 隐藏模型不会出现在导出的参考帧中。
- 锁定模型不能被误移动、误旋转或误缩放。

### 7.4 材质颜色调整

用户可以修改选中模型的静态材质颜色。

功能要求：

- 读取选中模型使用的材质槽。
- 优先展示材质名称；没有名称时使用可理解的默认命名。
- 支持通过颜色选择器修改基础颜色。
- 颜色修改实时应用到 3D 视口。
- 材质颜色覆盖需要保存到项目数据中。

MVP 暂不包含：

- 材质颜色关键帧动画。
- 贴图替换。
- 粗糙度、金属度、法线、透明度或 shader 编辑。

验收标准：

- 用户选中模型后可以修改至少一个材质颜色。
- 修改后的颜色能在视口中实时看到。
- 导出的项目 JSON 中包含材质颜色覆盖信息。

### 7.4A 骨骼控制

用户可以对带蒙皮骨骼的 `.glb` 模型进行静态 Pose 调整。

功能要求：

- 自动识别导入模型中的 `SkinnedMesh`、`Bone` 和骨架层级。
- 支持骨骼辅助线显示。
- 支持在左侧骨骼树和 3D 视口中选中骨骼。
- 支持 `FK` 模式，编辑选中骨骼旋转。
- 支持 `IK` 模式，通过指定根骨骼和末端骨骼创建骨链。
- 支持拖拽 IK 目标控制器，实时求解骨链 Pose。
- 支持每个模型保存当前静态 Pose 和 IK 链配置。

MVP 暂不包含：

- 骨骼关键帧时间线。
- 动画 clip 混合。
- 自动识别角色语义骨骼。
- 自动生成蒙皮骨架。

验收标准：

- 导入包含骨骼的 `.glb` 后，用户可以看到骨架层级。
- 用户可以在 `FK` 模式下旋转骨骼并实时看到模型 Pose 变化。
- 用户可以创建一条骨链，在 `IK` 模式下拖动目标控制器并驱动骨骼。

### 7.5 影视摄影机

用户可以创建和调整用于分镜预演的摄影机。

功能要求：

- 支持添加摄影机。
- 支持从当前视角创建摄影机。
- 支持重命名摄影机。
- 支持选择当前激活摄影机。
- 支持调整 FOV。
- 支持设置或更新摄影机 Look At 目标点。
- 在场景中显示摄影机辅助器。
- 支持常用景别预设：
  - 大远景
  - 远景
  - 中景
  - 近景
  - 特写
- 支持常用角度预设：
  - 平视
  - 俯拍
  - 仰拍
  - 侧拍
  - 近似过肩位

验收标准：

- 用户可以从当前视口创建摄影机。
- 用户可以切换到指定摄影机视角进行预览。
- 摄影机 FOV 和目标点变化能反映在视口预览中。

### 7.6 关键帧时间线

用户可以通过关键帧设计镜头运动。

MVP 关键帧通道：

- 摄影机位置。
- 摄影机目标点或旋转。
- 摄影机 FOV。
- 模型位置。
- 模型旋转。
- 模型缩放。

功能要求：

- 每个镜头都有独立时长。
- 时间线同时显示秒数和帧数。
- 用户可以在当前播放头位置添加关键帧。
- 用户可以移动播放头。
- 用户可以播放/暂停预览。
- 支持关键帧之间插值。
- 用户可以删除关键帧。

MVP 推荐限制：

- 第一版先使用线性插值。
- 缓入缓出等曲线预设放到后续版本。

验收标准：

- 用户可以创建至少两个摄影机关键帧，并预览二者之间的运动。
- 用户可以为模型变换添加关键帧。
- 导出的镜头 JSON 中包含关键帧数据。

### 7.7 镜头管理

用户可以在一个项目中管理多个分镜镜头。

功能要求：

- 支持创建镜头。
- 支持重命名镜头。
- 支持复制镜头。
- 支持删除镜头。
- 支持设置镜头时长。
- 支持为每个镜头记录描述、导演备注和 AI 提示词。
- 每个镜头可以关联一个主摄影机。

验收标准：

- 用户可以在一个项目中创建多个镜头。
- 切换镜头时能恢复该镜头的时间线和摄影机状态。
- 镜头元数据会出现在导出结果中。

### 7.8 导出

用户可以导出视觉参考和结构化参数。

必须支持的导出内容：

- 当前帧 PNG。
- 首帧 PNG。
- 尾帧 PNG。
- 单镜头参数 JSON。
- 完整项目 JSON。

参数 JSON 应包含：

- 项目元信息。
- 镜头元信息。
- 摄影机 FOV、位置、目标点和关键帧。
- 模型变换。
- 材质颜色覆盖。
- 画面尺寸、帧率和镜头时长。

验收标准：

- 导出的图片与所选摄影机和当前帧一致。
- 导出的 JSON 能用于重建镜头状态。
- JSON 可读、可版本化，并包含 schemaVersion。

## 8. 信息架构建议

### UI 参考

当前 UI 方向参考：

- `docs/assets/reference/ui-reference-3d-director.png`

视觉基调：

- 深色专业工作台，突出 3D 视口和空间网格。
- 中央视口占据主要面积，弱化装饰，强调镜头预演和空间判断。
- 左侧用于资产、对象和机位列表。
- 右侧用于快照管理、相机预览和当前选中对象属性。
- 底部使用浮动工具条承载高频操作，例如选择、角色/对象、相机、框选/适配、快照。
- 顶部仅保留产品名称、帮助和退出等全局信息，避免干扰创作视线。

交互原则：

- 视口优先：所有面板为视口服务，不抢占创作焦点。
- 参数可见：相机位置、注视点、FOV 等关键镜头参数应长期可见。
- 操作轻量：常用动作放在底部浮动工具条和右侧属性面板，减少层级菜单。
- 状态明确：当前选中对象、当前激活相机、是否可导出快照都要有清晰状态。

### 顶部工具栏

- 导入 GLB
- 添加摄影机
- 添加关键帧
- 导出
- 项目设置

### 左侧面板

- 镜头列表
- 对象/图层列表
- 搜索资产或对象
- 按对象、机位等类型分组

### 中央区域

- 3D 视口
- 摄影机预览叠层
- 变换控制器
- 地面网格和红/绿/蓝坐标轴
- 视角重置和方向辅助器

### 右侧面板

- 快照管理
- 相机预览缩略图
- 当前选中对象属性
- 模型变换
- 材质颜色控制
- 摄影机参数
- 镜头备注和提示词

### 底部区域

- 关键帧时间线
- 播放控制
- 当前时间/帧数显示
- 高频工具浮动条

第一阶段可先实现参考图中的基础布局，不强制实现完整时间线面板；时间线可以在后续里程碑展开到底部区域。

## 10. 技术方向建议

推荐前端 3D 技术栈：

- 使用 Three.js 实现浏览器端 3D 渲染。
- 使用 GLTFLoader 加载 `.glb` 模型。
- 使用 TransformControls 实现模型和摄影机变换控制。
- 使用 OrbitControls 实现视口导航。

推荐文件策略：

- 项目数据使用 JSON 保存。
- MVP 中 `.glb` 文件先作为外部资产处理。
- PNG 参考帧从 WebGL 画布导出。

## 11. 风险与未决问题

### 风险

- 大体积 `.glb` 模型可能影响浏览器性能。
- 部分 `.glb` 文件包含复杂材质或内嵌动画，MVP 需要明确处理边界。
- 摄影机目标点插值如果处理不好，镜头运动可能不自然。
- 时间线功能容易膨胀，需要控制第一版通道数量。

### 未决问题

- 导入的 `.glb` 文件是否复制到项目资产目录，还是仅引用本地路径？
- `.glb` 内嵌动画第一版是忽略，还是预留只读显示？
- 第一版是否每个镜头只允许一个激活摄影机？
- 导出的参考帧是否需要包含帧号、摄影机名称等叠层信息？
- 项目文件保存方式采用浏览器本地存储、本地文件，还是两者都支持？

## 13. 完成定义

一个功能只有满足以下条件才视为完成：

- 已更新或引用 PRD。
- 实现内容符合已确认范围。
- 验收标准已通过手动验证或自动化测试。
- UI 已在实际桌面视口尺寸下检查。
- 导出的 JSON 保持合法。
- 已记录已知限制。
